# 접근 방식 비교

## 개요

모바일에서 Claude Code를 원격으로 사용하는 세 가지 접근 방식을 비교한다.

- **1안 (HAPI)**: 오픈소스 프로젝트 [HAPI](https://github.com/tiann/hapi)를 사용
- **2안 (개인 서버 경유, 현재 사용)**: SSH 리버스 터널 + mosh + tmux 조합
- **3안 (Cloudflare Tunnel)**: Cloudflare 네트워크를 통해 SSH 노출

## 비교표

| 항목 | 1안: HAPI | 2안: 개인 서버 경유 | 3안: Cloudflare Tunnel |
|------|-----------|-------------------|----------------------|
| **모바일 UI** | 웹 UI (승인 버튼, 출력 렌더링) | 터미널 (mosh) | 터미널 (SSH) |
| **설정 복잡도** | `npx @twsxtd/hapi` 한 줄 | 서버 유저 + autossh + mosh | Cloudflare 계정/도메인 + 터널 |
| **폰에 설치할 앱** | 없음 (브라우저만) | Termius (+ Ntfy) | Termius (+ Ntfy) |
| **네트워크 전환 대응** | 릴레이가 처리 | mosh가 자동 복구 | 끊김 → 재접속 필요 |
| **mosh 지원** | 해당 없음 | 지원 (UDP) | 불가 (TCP만 프록시) |
| **푸시 알림** | 내장 | ntfy로 별도 구성 | ntfy로 별도 구성 |
| **승인 방식** | 원탭 버튼 | 터미널에서 직접 타이핑 | 터미널에서 직접 타이핑 |
| **음성 입력** | 지원 | 미지원 | 미지원 |
| **다중 에이전트** | 통합 UI에서 관리 | tmux window로 전환 | tmux window로 전환 |
| **다중 클라이언트 해상도** | 해당 없음 | linked session으로 독립 | linked session으로 독립 |
| **데이터 위치** | 맥북 로컬 유지 | 맥북 로컬 유지 | 맥북 로컬 유지 |

## 1안: HAPI

### 장점

- **모바일 최적화 UI**: 폰에서 터미널을 다루지 않아도 됨. 승인은 버튼 탭, 출력은 렌더링된 형태로 확인
- **설정 최소화**: `npx @twsxtd/hapi`로 바로 시작. 별도 도구 조합 불필요
- **올인원**: 네트워크 연결, 푸시 알림, 모바일 UI가 모두 내장
- **데이터 로컬 유지**: Happy와 달리 제3자 서버에 코드가 저장되지 않음

### 리스크

- **프로젝트 초기 단계**: 장기 유지보수 불확실. 초기 버그 존재
- **AGPL-3.0**: 회사 환경에서 라이선스 정책 확인 필요
- **Node.js 의존**: npx로 실행하므로 Node.js 필요
- **블랙박스 요소**: 내장 릴레이의 동작을 완전히 통제하기 어려움

### 현재 상태

초기 버그로 인해 사용하지 않음. 프로젝트가 안정화되면 재검토 예정.

## 2안: 개인 서버 경유 (현재 사용)

### 아키텍처

```
┌─ 맥북 ──────────────────────────────┐
│                                      │
│  tmux (caffeinate -s로 잠자기 방지)  │
│  ├─ Claude Code 세션들               │
│  └─ Claude 훅 → curl ntfy.sh        │
│                                      │
│  autossh (리버스 터널, launchd)      │
│  └─ -R 2222:localhost:22 → 서버     │
└──────────────────────────────────────┘
        │
        │ SSH 리버스 터널 (autossh)
        ▼
┌─ 개인 서버 ─────────────────────────┐
│                                      │
│  claude 유저 (프록시 전용, 셸 없음) │
│  localhost:2222 → 맥북:22           │
└──────────────────────────────────────┘
        ▲
        │ mosh (UDP)
        │
┌─ 스마트폰 ───────────────────────────┐
│                                      │
│  Termius: mosh → claude@서버         │
│  → 자동으로 맥북 tmux 세션 연결     │
└──────────────────────────────────────┘
```

### 장점

- **mosh 지원**: 네트워크 전환(와이파이 ↔ 셀룰러) 시 끊김 없이 자동 복구
- **안정성**: autossh, mosh, tmux는 오래 검증된 도구
- **독립성**: 특정 프로젝트에 의존하지 않아 하나가 죽어도 대체 가능
- **투명성**: 전체 구조를 이해하고 있어 문제 해결이 쉬움
- **보안**: 프록시 유저가 셸 접근 불가. 터널 포트는 localhost에서만 접근 가능
- **확장성**: 같은 서버에서 포트만 다르게 하여 다른 사용자 추가 가능

### 리스크

- **모바일 UX**: 폰에서 터미널을 다루는 건 본질적으로 불편
- **개인 서버 필요**: 공인 IP가 있는 서버가 있어야 함
- **관리 포인트**: autossh, 서버 유저, mosh, ntfy 훅 등 관리할 요소가 많음

## 3안: Cloudflare Tunnel

### 장점

- **무료 (도메인 비용만)**: Cloudflare Tunnel 자체는 무료
- **서버 불필요**: Cloudflare 네트워크가 중계
- **보안**: SSH 포트를 인터넷에 직접 열지 않음. Cloudflare Access로 MFA 추가 가능

### 리스크

- **mosh 불가**: TCP만 프록시하므로 UDP 기반 mosh 사용 불가
- **도메인 필요**: Cloudflare에 등록된 도메인이 필요 (연 ~$10)
- **Cloudflare 의존**: Cloudflare 서비스 장애 시 접속 불가

### 기각 사유

mosh를 지원하지 않아 네트워크 전환 시 연결이 끊기는 문제. 개인 서버가 있으므로 2안이 더 적합.

## 선택 이유

**2안(개인 서버 경유)을 선택한 이유:**

1. **mosh 지원**: 모바일 환경에서 네트워크 전환 시 끊김 없는 접속이 가장 중요한 요구사항
2. **개인 서버 보유**: 이미 개인 서버가 있어 추가 비용 없음
3. **HAPI 불안정**: 1안을 시도했으나 초기 버그로 실사용 불가
4. **Cloudflare Tunnel의 mosh 제한**: 3안은 mosh(UDP)를 지원하지 않음
5. **확장성**: 다른 사용자와 서버를 공유하기 쉬움

HAPI가 안정화되면 재검토할 예정이나, 현재 구성이 충분히 잘 동작함.

## 참고: Happy와의 차이

[Happy](https://github.com/slopus/happy)도 유사한 도구이지만, 중앙집중식 서버에 암호화된 데이터를 저장하는 방식이다. E2E 암호화(4계층 키 구조, AES-256)를 적용하여 서버가 복호화할 수 없지만, 코드가 제3자 서버를 경유한다는 점이 회사 보안 정책과 충돌할 수 있다. HAPI는 데이터가 로컬에 머무는 자체 호스팅 방식으로, 보안 관점에서 더 적합하다.
